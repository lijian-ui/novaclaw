# Rust UTF-8 字符串截断安全规则

## 问题
Rust 的 `&s[i..j]` 是**字节索引**，不是字符索引。中文字符占 3 字节、emoji 占 4 字节，
对包含非 ASCII 的字符串做字节切片会 panic。

## 正确做法
截取字符串长度时，必须使用字符安全的方式，禁止直接写 `&s[..n]`。

### 方式一：使用项目工具函数（推荐）
项目已有工具函数，路径：`backend/src/utils.rs`

```rust
/// 安全截断字符串：取前 max_len 个字符
pub fn safe_truncate(s: &str, max_len: usize) -> String {
    if s.len() > max_len {
        s.chars().take(max_len).collect()
    } else {
        s.to_string()
    }
}

/// 安全截断并追加省略号
pub fn safe_truncate_with_ellipsis(s: &str, max_len: usize) -> String {
    if s.len() > max_len {
        format!("{}...", s.chars().take(max_len).collect::<String>())
    } else {
        s.to_string()
    }
}
```

### 方式二：chars().take()
```rust
text.chars().take(80).collect::<String>()
```

### 方式三：char_indices 找边界
```rust
if let Some((end, _)) = text.char_indices().nth(80) {
    &text[..end]
}
```

## 适用范围
- 日志输出时截取长文本前缀
- 显示用的字符串截断
- 任何可能包含中文/非 ASCII 字符的字符串操作
