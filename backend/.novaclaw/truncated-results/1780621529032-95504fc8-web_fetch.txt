Microsoft Agent Framework (MAF) is our open-source SDK and runtime for building AI agents and multi-agent workflows, with the same concepts and APIs across .NET and Python. It gives you a clean programming model &#8211; chat clients, tools, MCP integrations, context providers, middleware, and multi-step workflows &#8211; so you can focus on agent logic instead of plumbing. MAF reached 1.0 GA on April 2, 2026, bringing the convergence of AutoGen and Semantic Kernel into a single, supported platform. Everything below builds on that 1.0 foundation.\nWhat&#8217;s new\nAgent Harness: production patterns, built in\nThe agent harness is the layer where model reasoning meets real execution: shell and filesystem access, human-in-the-loop approval flows, and context management across long-running sessions. With MAF, these patterns are now first-class and consistently built in from the get-go. stored in agent-file-memory/{session}/\nFileAccessProvider: general file access so that the agent can access and change any files it needs to operate on\nTodoProvider: lets the agent track work items (add, complete, remove, list) in session state for multi-step task management\nAgentModeProvider: supports &#8220;plan&#8221; vs &#8220;execute&#8221; operating modes so the agent can separate planning from action\nAgentSkillsProvider: skill discovery and execution from the file system, enabling modular capability injection\nBackgroundAgentsProvider: delegate subtasks to child agents running in parallel for fan-out orchestration\nWeb search: hosted web search tool out of the box (disable with DisableWebSearch)\nShell execution (.NET only): run shell commands via a sandboxed ShellExecutor\nMiddleware and customization:\nToolApprovalAgent: &#8220;don&#8217;t ask again&#8221;
