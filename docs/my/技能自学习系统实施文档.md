# 技能自学习系统实施文档

## 概述

让 LLM 在完成任务后，自动将经验、工作流、修复方案保存为技能（`SKILL.md`），供后续会话复用。实现后，每个会话都可以为技能库贡献新的知识，系统越用越强。

---

## 架构设计

```
用户消息 → Agent ReAct 循环
  │
  ├─ 每 N 轮工具迭代触发「后台审查」
  │    └─ 后台 LLM 调用（独立分叉）
  │         ├─ 检查是否发现了新工作流
  │         ├─ 检查已加载的技能是否需要更新
  │         └─ 调用 skill_manage 创建/更新技能
  │
  └─ 用户主动要求「记住这个」
       └─ 调用 skill_manage 创建技能
```

/// 注册 skill_manage 和 skill_self_learn 工具
pub fn register_skill_tools(registry: &Arc<crate::tools::registry::ToolRegistry>) {
    let registry = registry.clone();
    
    // 异步执行注册
    tokio::task::block_in_place(|| {

async fn register_skill_manage(registry: &Arc<crate::tools::registry::ToolRegistry>) {
    registry.register(crate::tools::types::ToolDef {
        name: "skill_manage".to_string(),
        description: r#"Create, edit, patch, or delete skills.

Use this to save reusable workflows as skills so they can be reused in future sessions.
Create when: complex task succeeded (5+ calls), errors overcome, user-corrected approach worked,
non-trivial workflow discovered, or user asks you to remember a procedure.
After difficult/iterative tasks, offer to save as a skill.
If you used a skill and found it outdated, incomplete, or wrong, patch it immediately.

Actions:
- "create": Create a new skill. Provide name, content (with YAML frontmatter + markdown body), and optional category.
- "edit": Full rewrite of a skill's content.
- "patch": Targeted text replacement (old_string → new_string). Use this for small fixes.
- "delete": Remove a skill."#.to_string(),
        parameters: json!({
            "type": "object",
            "properties": {
                "action": {
                    "type": "string",
                    "enum": ["create", "edit", "patch", "delete"],
                    "description": "What to do with the skill"
                },
                "name": {
                    "type": "string",
                    "description": "Skill name (lowercase, hyphens, max 64 chars)"
                },
                "content": {
                    "type": "string",
                    "description": "Full SKILL.md content with YAML frontmatter (required for create/edit)"
                },
                "old_string": {
                    "type": "string",
                    "description": "Text to replace (required for patch)"
                },
                "new_string": {
                    "type": "string",
                    "description": "Replacement text (required for patch)"
                }
            },
            "required": ["action", "name"]
        }),
        handler: Arc::new(move |args: serde_json::Value, _chunk_tx| -> Result<String, String> {
            let action = args["action"].as_str().ok_or("Missing 'action'")?.to_string();
            let name = args["name"].as_str().ok_or("Missing 'name'")?.to_string();
            
            let state = APP_STATE.blocking_read();
            let skills_dir = state.config.skills_dir();
            let loader = SkillsLoader::new(&skills_dir);
            drop(state);
            
            match action.as_str() {
                "create" => {
                    let content = args["content"].as_str().ok_or("Missing 'content' for create")?;
                    // 验证格式
                    if !content.contains("---") {
                        return Err("Content must include YAML frontmatter (---)".to_string());
                    }
                    if !content.contains("name:") {
                        return Err("Frontmatter must include 'name:' field".to_string());
                    }
                    if !content.contains("description:") {
                        return Err("Frontmatter must include 'description:' field".to_string());
                    }
                    
                    // 创建技能目录和 SKILL.md
                    let skill_dir = skills_dir.join(&name);
                    std::fs::create_dir_all(&skill_dir)
                        .map_err(|e| format!("创建目录失败: {}", e))?;
                    
                    // 写入 SKILL.md
                    let skill_path = skill_dir.join("SKILL.md");
                    // 如果文件已存在则追加内容
                    let final_content = if skill_path.exists() {
                        let existing = std::fs::read_to_string(&skill_path)
                            .unwrap_or_default();
                        format!("{}\n\n---\n\n{}", existing.trim(), content)
                    } else {
                        content.to_string()
                    };
                    std::fs::write(&skill_path, &final_content)
                        .map_err(|e| format!("写入失败: {}", e))?;
                    
                    Ok(json!({"success": true, "message": format!("技能 '{}' 已创建", name)}).to_string())
                }
                "edit" => {
                    let content = args["content"].as_str().ok_or("Missing 'content' for edit")?;
                    let skill_path = skills_dir.join(&name).join("SKILL.md");
                    if !skill_path.exists() {
                        return Err(format!("技能 '{}' 不存在", name));
                    }
                    std::fs::write(&skill_path, content)
                        .map_err(|e| format!("写入失败: {}", e))?;
                    Ok(json!({"success": true, "message": format!("技能 '{}' 已更新", name)}).to_string())
                }
                "patch" => {
                    let old = args["old_string"].as_str().ok_or("Missing 'old_string' for patch")?;
                    let new = args["new_string"].as_str().ok_or("Missing 'new_string' for patch")?;
                    let skill_path = skills_dir.join(&name).join("SKILL.md");
                    if !skill_path.exists() {
                        return Err(format!("技能 '{}' 不存在", name));
                    }
                    let content = std::fs::read_to_string(&skill_path)
                        .map_err(|e| format!("读取失败: {}", e))?;
                    if !content.contains(old) {
                        return Err(format!("未在技能 '{}' 中找到匹配文本", name));
                    }
                    let updated = content.replace(old, new);
                    std::fs::write(&skill_path, &updated)
                        .map_err(|e| format!("写入失败: {}", e))?;
                    Ok(json!({"success": true, "message": format!("技能 '{}' 已修补", name)}).to_string())
                }
                "delete" => {
                    let skill_dir = skills_dir.join(&name);
                    if !skill_dir.exists() {
                        return Err(format!("技能 '{}' 不存在", name));
                    }
                    std::fs::remove_dir_all(&skill_dir)
                        .map_err(|e| format!("删除失败: {}", e))?;
                    Ok(json!({"success": true, "message": format!("技能 '{}' 已删除", name)}).to_string())
                }
                _ => Err(format!("未知操作: {}", action)),
            }
        }),
    });
}

async fn register_self_learn_trigger(registry: &Arc<crate::tools::registry::ToolRegistry>) {
    registry.register(crate::tools::types::ToolDef {
        name: "skill_self_review".to_string(),
        description: r#"Internal tool: After a complex task (5+ tool calls), review the conversation and create or update skills based on what was learned. Called automatically by the system. Do not call manually."#.to_string(),
        parameters: json!({
            "type": "object",
            "properties": {
                "conversation_summary": {
                    "type": "string",
                    "description": "Summary of what happened in this conversation"
                },
                "tool_call_count": {
                    "type": "integer",
                    "description": "Number of tool calls made"
                }
            },
            "required": ["conversation_summary", "tool_call_count"]
        }),
        handler: Arc::new(|args, _chunk_tx| -> Result<String, String> {
            let summary = args["conversation_summary"].as_str().unwrap_or("");
            let tool_count = args["tool_call_count"].as_i64().unwrap_or(0);
            
            if tool_count < 5 {
                return Ok(json!({"action": "none", "reason": "Too few tool calls to warrant a skill"}).to_string());
            }
            
            // 生成技能建议
            Ok(json!({
                "action": "suggest",
                "reason": format!("Complex task completed ({} tool calls)", tool_count),
                "summary": summary,
                "suggestion": "Review the conversation and decide if a new skill should be created or an existing one updated."
            }).to_string())
        }),
    });
}

### Step 3: 后台审查系统

#### 文件: `backend/src/agent/skill_review.rs`（新建）

```rust
use crate::agent::session::AgentMessage;
use std::sync::atomic::{AtomicBool, Ordering};
use std::sync::Arc;

/// 后台技能审查配置
pub struct SkillReviewConfig {
    /// 每 N 次工具迭代触发一次审查（0=禁用）
    pub nudge_interval: u32,
}

impl Default for SkillReviewConfig {
    fn default() -> Self {
        Self { nudge_interval: 10 }
    }
}

/// 后台技能审查器
pub struct SkillReviewer {
    /// 距离上次审查的工具迭代计数
    iters_since_last_review: u32,
    /// 审查配置
    config: SkillReviewConfig,
    /// 是否正在审查中
    reviewing: Arc<AtomicBool>,
}

impl SkillReviewer {
    pub fn new(config: SkillReviewConfig) -> Self {
        Self {
            iters_since_last_review: 0,
            config,
            reviewing: Arc::new(AtomicBool::new(false)),
        }
    }

    /// 每次工具调用后调用，判断是否需要触发审查
    pub fn on_tool_call(&mut self) -> bool {
        if self.config.nudge_interval == 0 {
            return false;
        }
        self.iters_since_last_review += 1;
        if self.iters_since_last_review >= self.config.nudge_interval {
            self.iters_since_last_review = 0;
            true
        } else {
            false
        }
    }

    /// 重置计数器（skill_manage 被调用时）
    pub fn reset(&mut self) {
        self.iters_since_last_review = 0;
    }

    /// 启动后台审查（异步，不阻塞主循环）
    pub fn spawn_review(
        &self,
        messages: Vec<AgentMessage>,
        tool_call_count: u32,
    ) {
        if self.reviewing.load(Ordering::Relaxed) {
            return; // 已有审查进行中
        }
        self.reviewing.store(true, Ordering::Relaxed);

        let reviewing = self.reviewing.clone();
        tokio::spawn(async move {
            // 这里可以调用 LLM 进行审查
            // 但为了简化实现，首期只做计数记录
            tracing::info!(
                "[SkillReview] {} 次工具调用后触发技能审查",
                tool_call_count
            );
            reviewing.store(false, Ordering::Relaxed);
        });
    }
}
```

```markdown

name: my-skill-name
description: Brief one-line description of what this skill does
version: 1.0.0

# Skill Title

## Overview
Brief description of when to use this skill.

## Steps
1. Step one with commands
2. Step two with commands
3. ...

## Commands
```bash
# Example commands
command --option value
```

## Notes
- Pitfalls to avoid
- Important details
- Configuration requirements
```

## 一期实现建议

首期可以只实现 **Step 1 和 Step 2**：

1. 注册 `skill_manage` 工具（CRUD）
2. 在 system prompt 中告诉 LLM 可以创建/更新技能
3. 依赖 LLM 自主判断是否创建技能（靠 prompt 引导，不要后台审查）

等跑通了再考虑加 Step 3 的后台自动审查。

---

## 数据流

```
完成复杂任务后
  │
  ├─ LLM 自主判断（受 prompt 引导）
  │    └─ 调用 skill_manage(action='create', name=..., content=...)
  │         → 创建 skills/技能名/SKILL.md
  │         → 下次会话自动出现在 <available_skills> 中
  │
  ├─ 用户主动要求
  │    └─ "把这个保存为技能"
  │         → LLM 调用 skill_manage
  │
  └─ 技能需要修补
       └─ "这个技能少了一步"
            → LLM 调用 skill_manage(action='patch', ...)
```
