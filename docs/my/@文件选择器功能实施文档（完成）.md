# @ 文件选择器功能实施文档

## 功能概述

在聊天输入框中输入 `@` 时，弹出当前工作目录的文件选择下拉菜单，用户可选择文件作为对话上下文。提交消息时，后端自动将 `@路径` 替换为实际文件内容。

---

## 数据流

```
用户输入 "@" → 前端检测到 @ 前缀
  ↓
弹出下拉框，发起文件查询请求
  ↓
后端返回当前目录/匹配的文件列表
  ↓
用户选择 → 插入 "@路径 " 到输入框
  ↓
提交消息 → 后端展开 @路径 为文件内容
```

```rust
use serde::Deserialize;
use std::path::Path;
use tokio::fs;

/// @-mention 文件查询请求
#[derive(Deserialize)]
struct MentionQuery {
    workspace: Option<String>,  // 工作目录
    query: String,              // @ 后面的查询内容（空=浏览根目录）
}

/// @-mention 展开请求
#[derive(Deserialize)]
struct ExpandMentionReq {
    content: String,  // 含 @路径 的消息原文
    workspace: Option<String>,
}

/// 查询目录文件列表
async fn list_mentions(Json(req): Json<MentionQuery>) -> Json<serde_json::Value> {
    let state = APP_STATE.read().await;
    let workspace = req.workspace.as_deref()
        .unwrap_or("") // 使用当前工作目录或配置文件中的默认路径
        .to_string();
    let workspace_path = if workspace.is_empty() {
        state.config.workspace_dir()
    } else {
        std::path::PathBuf::from(&workspace)
    };
    drop(state);

    if !workspace_path.exists() {
        return Json(serde_json::json!({
            "success": false,
            "message": "工作目录不存在"
        }));
    }

    // 解析查询路径
    let query = req.query.trim();
    let search_dir = if query.is_empty() {
        workspace_path.clone()
    } else {
        // 先尝试完整路径
        let full_path = workspace_path.join(query);
        if full_path.is_dir() {
            // 浏览模式：列出该目录内容
            full_path
        } else {
            // 搜索模式：返回 workspace_path，后续模糊匹配
            workspace_path.clone()
        }
    };

    let mut entries = Vec::new();
    let mut read_dir = match fs::read_dir(&search_dir).await {
        Ok(d) => d,
        Err(_) => return Json(serde_json::json!({
            "success": false,
            "message": "读取目录失败"
        })),
    };

    // 是否在搜索模式
    let is_search = if query.is_empty() {
        false
    } else {
        let full_path = workspace_path.join(query);
        !full_path.is_dir()
    };
    let search_term = if is_search {
        query.split('/').last().unwrap_or(query).to_lowercase()
    } else {
        String::new()
    };

    while let Ok(Some(entry)) = read_dir.next_entry().await {
        let name = entry.file_name().to_string_lossy().to_string();
        // 忽略隐藏文件和常见忽略目录
        if name.starts_with('.') || name == "node_modules" || name == "target" || name == ".git" {
            continue;
        }
        let path = entry.path();
        let is_dir = path.is_dir();
        let rel_path = pathdiff::diff_paths(&path, &workspace_path)
            .unwrap_or_else(|| path.clone())
            .to_string_lossy()
            .to_string();

        if is_search {
            // 搜索模式：文件名匹配
            if !name.to_lowercase().contains(&search_term) {
                continue;
            }
        }

        entries.push(serde_json::json!({
            "name": name,
            "path": rel_path,
            "is_dir": is_dir,
        }));
    }

    // 目录排前面，然后按名称排序
    entries.sort_by(|a, b| {
        let a_dir = a["is_dir"].as_bool().unwrap_or(false);
        let b_dir = b["is_dir"].as_bool().unwrap_or(false);
        if a_dir != b_dir {
            b_dir.cmp(&a_dir)
        } else {
            a["name"].as_str().unwrap_or("").cmp(b["name"].as_str().unwrap_or(""))
        }
    });

    Json(serde_json::json!({
        "success": true,
        "data": entries
    }))
}

/// 展开 @ 引用为文件内容
async fn expand_mentions(Json(req): Json<ExpandMentionReq>) -> Json<serde_json::Value> {
    let state = APP_STATE.read().await;
    let workspace = req.workspace.as_deref()
        .unwrap_or("")
        .to_string();
    let workspace_path = if workspace.is_empty() {
        state.config.workspace_dir()
    } else {
        std::path::PathBuf::from(&workspace)
    };
    drop(state);

    // 用正则匹配所有 @路径 引用
    let re = regex::Regex::new(r"(?:(?<=\s)|(?<=^))@([^\s，。、；：）\)]+)").unwrap();
    let result = re.replace_all(&req.content, |caps: &regex::Captures| {
        let path_str = &caps[1];
        let full_path = workspace_path.join(path_str);
        let normalized = full_path.to_string_lossy().to_string();

        if full_path.is_dir() {
            // 目录：列出条目数
            let count = std::fs::read_dir(&full_path).map(|d| d.count()).unwrap_or(0);
            format!("<directory path=\"{}\" entries=\"{}\" />", path_str, count)
        } else if full_path.is_file() {
            // 文件：读取内容
            match std::fs::read_to_string(&normalized) {
                Ok(content) => {
                    format!("<file path=\"{}\">\n{}\n</file>", path_str, content)
                }
                Err(_) => format!("@{}（读取失败）", path_str),
            }
        } else {
            format!("@{}（文件不存在）", path_str)
        }
    });

    Json(serde_json::json!({
        "success": true,
        "data": result.to_string()
    }))
}
```

### Step 3: 前端 — 添加 API 调用函数

#### 文件: `src/hooks/useApi.ts`

在 `useApi` 的返回对象中添加两个方法：

```typescript
import { API_BASE } from '@/hooks/useApi'

// 放在文件中（export 前）
export async function queryMentions(
  workspace: string | undefined,
  query: string
): Promise<{ name: string; path: string; is_dir: boolean }[]> {
  try {
    const res = await fetch(`${API_BASE}/mentions`, {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({ workspace, query }),
    })
    const body = await res.json()
    return body.success ? (body.data || []) : []
  } catch {
    return []
  }
}

export async function expandMentions(
  content: string,
  workspace?: string
): Promise<string> {
  try {
    const res = await fetch(`${API_BASE}/mentions/expand`, {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({ content, workspace }),
    })
    const body = await res.json()
    return body.success ? (body.data || content) : content
  } catch {
    return content
  }
}
```

```typescript
const handleInputChange = useCallback((e: React.ChangeEvent<HTMLTextAreaElement>) => {
  const value = e.target.value
  
  // 检测 @mention
  const match = value.match(/(?:^|\s)@([^\s]*)$/)
  if (match) {
    const query = match[1]
    setMentionQuery(query)
    setMentionOpen(true)
    setMentionIndex(0)
    // 调用 API 获取文件列表
    void queryMentions(workspacePath, query).then(items => {
      setMentionItems(items)
    })
  } else {
    setMentionOpen(false)
  }
  
  // 原有输入处理逻辑...
  setInput(value)
}, [workspacePath])
```

```typescript
const handleKeyDown = useCallback((e: React.KeyboardEvent<HTMLTextAreaElement>) => {
  if (mentionOpen && mentionItems.length > 0) {
    if (e.key === 'ArrowDown') {
      e.preventDefault()
      setMentionIndex(i => Math.min(i + 1, mentionItems.length - 1))
      return
    }
    if (e.key === 'ArrowUp') {
      e.preventDefault()
      setMentionIndex(i => Math.max(i - 1, 0))
      return
    }
    if (e.key === 'Enter' && mentionItems[mentionIndex]) {
      e.preventDefault()
      insertMention(mentionItems[mentionIndex])
      return
    }
    if (e.key === 'Escape') {
      setMentionOpen(false)
      return
    }
  }
  // 原有键盘处理逻辑...
}, [mentionOpen, mentionItems, mentionIndex, input, workspacePath])
```

**4.4 插入选择**

```typescript
const insertMention = useCallback((item: { name: string; path: string; is_dir: boolean }) => {
  const input = inputRef.current
  if (!input) return
  
  const cursorPos = input.selectionStart ?? input.value.length
  const before = input.value.slice(0, cursorPos)
  const after = input.value.slice(cursorPos)
  
  // 找到最后一个 @ 的位置
  const atPos = before.lastIndexOf('@')
  if (atPos === -1) return
  
  const mentionText = item.is_dir ? `@${item.path}/` : `@${item.path} `
  const newValue = before.slice(0, atPos) + mentionText + after
  
  setInput(newValue)
  setMentionOpen(false)
  
  // 将光标移到插入内容后面
  requestAnimationFrame(() => {
    const pos = atPos + mentionText.length
    input.setSelectionRange(pos, pos)
    input.focus()
  })
}, [])
```

## 注意事项

1. **性能**：在非常大的工作目录中，文件列表可能较慢。可考虑添加异步流式查询或缓存。
2. **安全**：`expand_mentions` 中需要做路径遍历防护，确保 `@../../etc/passwd` 这类路径不会读取到工作目录之外的文件。
3. **路径遍历防护示例**：

```rust
// 检查展开路径是否在工作目录内
let canonical_workspace = workspace_path.canonicalize().ok()?;
let canonical_full = full_path.canonicalize().ok()?;
if !canonical_full.starts_with(&canonical_workspace) {
    return format!("@{}（路径不允许）", path_str);
}
```

4. **引用文件编码**：如果文件是二进制或非 UTF-8 编码，读取时会失败，需要做 `lossy` 降级处理。
5. **前端防抖**：快速输入时，建议对 `queryMentions` 调用做 300ms 防抖，避免频繁请求。
