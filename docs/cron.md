# 定时任务

jeeves 支持两种方式创建定时任务。
1. **自然语言创建**：直接向 Agent 描述需求，例如：每天18:00帮我发送日报，Agent 会自动解析并创建定时任务
2. **手动配置**：通过前端定时任务配置页面手动创建

## 任务管理

### 创建任务

**方式一：自然语言创建**

直接向Agent 发送请求：
```
每天18:00帮我发送日报```

Agent 会自动解析时间和任务内容，创建相应的定时任务。
**方式二：手动创建**

通过前端配置页面的API 创建。
```json
{
  "name": "每日报告",
  "schedule": "0 18 * * *",
  "payload": "帮我生成今日报告",
  "session_id": "可选的会话ID",
  "enabled": true
}
```

### Cron 表达式格式
标准的5 字段 Cron 表达式：

```
┌───────────── 分钟 (0-59)
│┌─────────── 小时 (0-23)
││┌───────── 日期 (1-31)
│││┌─────── 月份 (1-12)
││││┌───── 星期 (0-7)
│││││* * * * *
```

### 常用表达式
| 表达式| 说明 |
|--------|------|
| `0 * * * *` | 每小时整点|
| `0 8 * * *` | 每天早上 8 点|
| `0 0 * * *` | 每天午夜 |
| `0 0 * * MON` | 每周一午夜 |
| `0 0 1 * *` | 每月 1 日|

## 任务状态
### 状态说明
| 状态| 说明 |
|------|------|
| `enabled` | 已启用，按计划执行|
| `disabled` | 已禁用，不会执行 |
| `running` | 正在执行中|

### 切换状态
```bash
POST /api/cron-jobs/{id}/toggle
```

## 任务执行

### 立即执行

可以手动触发任务立即执行。
```bash
POST /api/cron-jobs/{id}/run
```

### 执行记录

每个任务都会记录执行历史：- 上次执行时间
- 下次执行时间
- 执行次数
- 执行结果

## 任务日志

任务执行日志存储在`logs/` 目录中：

```bash
GET /api/logs/tasks/{task_id}
```

## 任务配置示例

### 示例 1：每日数据备│
```json
{
  "name": "每日数据备份",
  "schedule": "0 2 * * *",
  "payload": "帮我备份今天的重要数据",
  "enabled": true
}
```

### 示例 2：定时提醒
```json
{
  "name": "工作提醒",
  "schedule": "0 9,14,18 * * *",
  "payload": "提醒我休息一下",
  "enabled": true
}
```

### 示例 3：每周报告
```json
{
  "name": "周报生成",
  "schedule": "0 18 * * FRI",
  "payload": "帮我生成本周工作报告",
  "enabled": true
}
```

## 最佳实践
### 1. 避免重复

检查是否已有类似任务，避免重复执行。
### 2. 设置合理时间

避免在系统负载高峰期执行任务。
### 3. 监控执行状态
定期检查任务执行状态和日志。
### 4. 清理历史日志

定期清理不再需要的任务日志。
## 任务存储

任务配置存储在`cron/` 目录中，支持持久化保存。
