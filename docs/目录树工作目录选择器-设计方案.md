# 目录树工作目录选择器设计文档

## 1. 背景

### 1.1 问题

NovaClaw 支持用户自定义工作目录，当前有两种设置方式：

| 方式 | 环境 | 限制 |
|---|---|---|
| Tauri 原生对话框 | Tauri 桌面端（Win/Mac/Linux） | 仅桌面端可用 |
| 手动输入路径 | 所有环境 | 需要用户知道完整路径，体验不够直观 |

前后端分离的 Web 模式下（包括 Linux 端），浏览器 `<input webkitdirectory>` 存在安全限制，无法获取文件夹的完整绝对路径，导致"打开文件夹"按钮只能获取文件夹名称。

### 1.2 目标

在 Web 端实现一个**可视化的目录树选择器**，让用户可以通过浏览目录结构来选择和切换工作目录，无需手动输入路径。

---

## 2. 架构设计

### 2.1 数据流

```
用户操作                  前端                       后端
─────────           ────────────              ────────────
点击工作区名称  → 弹出自定义下拉菜单
                  点击"浏览目录"
                  打开目录树面板
                  展开目录节点  ──→  POST /api/files/list-tree { path }
                                  ←──  { subdirs: ["dir1", "dir2", ...] }
                  选择目录节点  ──→  更新 workspacePath 状态
                                  ──→  传递到 ChatPanel
                                  ──→  传递到 FileExplorer（刷新文件列表）
                                  ──→  下次对话携带 workspace 参数到后端
                                  ──→  LLM 感知新工作目录
```

### 2.2 涉及组件

| 组件 | 角色 |
|---|---|
| `App.tsx` | 持有 `workspacePath` 状态，分发给子组件 |
| `ChatPanel.tsx` | 触发目录选择，传递 workspace 给后端 |
| `FileExplorer.tsx` | 接收到新路径后刷新文件列表 |
| 后端 `POST /api/files/list-tree` | 接收任意路径，返回子目录列表 |

## 3. 后端 API 设计

### 3.1 `POST /api/files/list-tree`

**请求**：
```json
{
    "path": "/home/user"
}
```

**响应**：
```json
{
    "success": true,
    "data": {
        "current_path": "/home/user",
        "parent_path": "/home",
        "subdirs": [
            { "name": "projects", "path": "/home/user/projects" },
            { "name": "downloads", "path": "/home/user/downloads" },
            { "name": "Documents", "path": "/home/user/Documents" }
        ]
    }
}
```

**错误响应**：
```json
{
    "success": false,
    "message": "Directory not found: /not/exist"
}
```

### 3.2 关键实现逻辑

```rust
// REST 端点（在 server/routes/files.rs 中新增）
async fn list_tree(Json(req): Json<ListTreeReq>) -> Json<serde_json::Value> {
    let path = Path::new(&req.path);
    if !path.exists() || !path.is_dir() {
        return Json(json!({ "success": false, "message": "目录不存在" }));
    }

    let mut subdirs = Vec::new();
    for entry in std::fs::read_dir(path).map_err(|e| e.to_string())? {
        let entry = entry.map_err(|e| e.to_string())?;
        if entry.file_type()?.is_dir() {
            let name = entry.file_name().to_string_lossy().to_string();
            // 跳过隐藏目录
            if name.starts_with('.') { continue; }
            subdirs.push(json!({
                "name": name,
                "path": entry.path().to_string_lossy()
            }));
        }
    }

    subdirs.sort_by(|a, b| a["name"].as_str().cmp(&b["name"].as_str()));

    Json(json!({
        "success": true,
        "data": {
            "current_path": req.path,
            "parent_path": path.parent().map(|p| p.to_string_lossy()),
            "subdirs": subdirs
        }
    }))
}
```

> **注意**：这是一个纯后端 API，不依赖 Tauri 或任何桌面端能力，所有环境（包括纯浏览器）都能使用。

### 3.3 安全考量

- 路径穿越防护：需要校验 path 参数防止遍历系统敏感目录
- 可选白名单：允许配置可浏览的根路径范围（如只允许 `/home`、`/` 等）
- 权限检查：跳过用户无权限读取的目录（读取失败时静默跳过而非报错）

### 4.2 目录树组件（TreeBrowser）

**Props**：
```
interface TreeBrowserProps {
    initialPath?: string          // 初始显示的路径
    onSelect: (path: string) => void  // 选中回调
    onCancel: () => void          // 取消回调
}
```

**状态管理**：
```
nodeStates: Map<string, {
    expanded: boolean           // 是否已展开

currentPath: string             // 当前选中路径
```

### 4.3 懒加载机制

- 默认只展开根目录（`/` 或用户初始路径）
- 点击目录左侧的箭头 ▶ 时，才加载该目录的子目录列表
- 已展开的目录缓存子目录列表，避免重复请求
- 加载中显示骨架屏或 loading spinner

## 5. 与现有系统的集成

### 5.1 状态管理

`workspacePath` 状态上升至 `App.tsx`：

```typescript
// App.tsx
const [workspacePath, setWorkspacePath] = useState('')

// 传给 ChatPanel
<ChatPanel
    workspacePath={workspacePath}
    onWorkspacePathChange={setWorkspacePath}
/>

// 传给 FileExplorer
<FileExplorer
    customPath={workspacePath}
/>
```

### 5.2 与 ChatPanel 的联动

当用户通过目录树选择一个路径后：

1. `TreeBrowser.onSelect(path)` 被调用
2. `App.tsx` 中 `setWorkspacePath(path)`
3. `ChatPanel` 感知到 `workspacePath` 更新
4. `ChatPanel` 工作区名称显示为选中目录的文件夹名
5. 下次发送消息时，`startChatStream` 携带新的 `workspace` 参数
6. `FileExplorer` 感知到 `customPath` 更新，自动刷新文件列表

### 5.3 与后端的联动

后端 `POST /api/chat/stream` 接收到 `workspace` 参数后：

1. `AgentSession.workspace` = 新路径
2. `SystemPromptBuilder` 输出 `Working directory: /home/user/projects/novaclaw`
3. LLM 知道当前工作目录
4. `resolve_path()` 将相对路径解析到新工作目录
5. `grep` / `glob` / `read_file` / `write_file` 等工具在新目录下操作

## 6. 边界情况处理

| 场景 | 处理方式 |
|---|---|
| 空目录 | 显示 "(empty directory)" |
| 无权限目录 | 静默跳过，不报错 |
| 路径不存在 | 显示 "目录不存在" 提示 |
| 根目录 `/` | 允许作为起始浏览点，但选择时弹出确认 |
| Windows 盘符 | 起始路径改为 `C:\`，列出所有可用驱动器 |
| 目录包含大量子目录（1000+） | 仅首屏加载，支持滚动加载 |
| 网络请求失败 | 显示重试按钮，保留已展开的缓存 |
| 用户选择后快速切换 | 防抖处理，忽略中间状态 |
| `node_modules`、`.git` 等 | 自动隐藏，减少干扰 |

## 8. 备选方案

如果目录树选择器开发成本过高，可以考虑降级方案：

### 方案 A：快速路径预设

```typescript
const PRESET_PATHS = {
    "项目目录": "c:/project/novaclaw",
    "工作区": "C:/Users/xxx/AppData/Local/novaclaw/workspace",
    "文档": "C:/Users/xxx/Documents",
}
```

用户点击预设按钮一键切换，适合经常在几个固定目录间切换的场景。

### 方案 B：从系统目录自动发现

扫描常见的项目存放位置（`~/projects/`、`~/workspace/`、`~/code/` 等），在菜单中列出找到的项目目录供用户选择。

### 方案 C：结合 Tauri 端能力

在 Tauri 桌面端继续使用原生对话框（已有），Web 端使用路径预设 + 文本输入的组合方案。
