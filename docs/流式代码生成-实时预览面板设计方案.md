# 流式代码生成 — 实时预览面板设计方案

## 1. 背景

### 1.1 问题

当前 NovaClaw 的 AI Agent 生成代码时，流程是：

1. LLM 流式输出代码到聊天框（`text_chunk` 事件）
2. 生成完毕后调用 `write_file` 一次性写入磁盘
3. 用户只能在聊天框中看到最终代码

缺乏**实时预览**能力——代码在生成过程中用户看不到 diff 对比，也无法提前感知改动范围。

### 1.2 目标

实现一个**实时代码预览面板**，在 LLM 生成代码的同时，实时渲染出差分对比（diff），让用户在代码落盘前就能看到改动内容。

---

## 2. 数据流

### 2.1 当前架构（无预览）

```
LLM 流式生成代码 → SSE text_chunk → 聊天框渲染
                                         ↓ 生成完毕
                                    write_file 调用 → 磁盘写入 → 文件编辑器刷新
```

### 2.2 目标架构（有预览）

```
LLM 流式生成代码 → SSE text_chunk
                    ├──→ 聊天框渲染（已有）
                    └──→ 实时预览面板
                          ├── 解析 text_chunk 中的文件路径和内容
                          ├── 生成与原文件的 diff 对比
                          └── 实时更新 diff 视图
                                         ↓ 生成完毕
                                    write_file 调用 → 磁盘写入
                                         ↓
                                    预览面板标记为"已保存"
```

---

## 3. 涉及组件

### 3.1 后端改动（无/极少）

后端**不需要改**，现有 SSE 流已经包含了足够的信息：

```
type: "chunk"
data: "```typescript\n// src/app.ts\nfunction hello() {\n  console.log(\"world\")\n}\n```"
```

只需要在 `text_chunk` 事件中额外携带**文件名信息**——如果 AI 在生成代码前先说了 `write_file path="xxx"`，后续生成的文本片段应该关联到这个文件。

**可选优化**：在 `AgentStep` 中增加一个 `file_path` 字段：

```typescript
interface AgentStep {
  step_type: "text_chunk" | "tool_call" | "tool_result" | ...
  content: string
  file_path?: string  // 新增：当前生成的代码属于哪个文件
}
```

### 3.2 前端核心组件

| 组件 | 角色 |
|---|---|
| **StreamingDiffPanel** | 流式 diff 预览容器，管理多个文件的 diff 视图 |
| **DiffView** | 单个文件的 diff 渲染（使用 monaco-diff 或 react-diff-viewer） |
| **FileTabs** | 多个待写入文件之间的切换标签 |

## 4. UI 布局

### 4.1 预览面板位置

```
┌──────────────┬──────────────────────┬──────────────┐
│              │                      │  📁 FileExplorer │
│  聊天面板     │  (主编辑区)           │              │
│  ChatPanel   │  FileEditor          │  流式预览面板  │
│              │                      │  Streaming    │
│              │                      │  DiffPanel    │
│              │                      │              │
└──────────────┴──────────────────────┴──────────────┘
```

预览面板作为 **FileExplorer 上方的浮动区域**，或者作为独立的**底栏/侧栏标签**。

### 4.2 交互状态

```
┌─────────────────────────────────────┐
│  📄 流式生成预览                      │
│  ┌───┬───┐                          │
│  │app.ts│ │auth.ts│                  │  ← 文件标签（多个文件切换）
│  └───┴───┘                          │
│  ┌─────────────────────────────────┐ │
│  │ - old line                       │ │  ← 删除行（红色）
│  │ + new line                       │ │  ← 新增行（绿色）
│  │   unchanged line                 │ │  ← 未改动行（灰色）
│  │ ... (更多行正在生成中...)         │ │  ← 正在流式输出（动画闪烁）
│  └─────────────────────────────────┘ │
│  [ 等待写入磁盘... ]                 │  ← 状态指示
└─────────────────────────────────────┘
```

### 4.3 状态指示

| 状态 | 显示 |
|---|---|
| **生成中** | 文件名标签显示 `●` 蓝点 + 逐行动画展开 |
| **已写入** | 标签显示 `✓` 绿勾 + 保留 diff 视图 | 
| **写入失败** | 标签显示 `✗` 红叉 + 错误信息 |
| **用户已关闭** | 清除该文件的预览状态 |

## 5. Diff 引擎选择

### 方案 A：react-diff-viewer（推荐）

```bash
npm install react-diff-viewer-continued
```

```tsx
import ReactDiffViewer from 'react-diff-viewer-continued'

function DiffView({ oldCode, newCode }: { oldCode: string; newCode: string }) {
  return (
    <ReactDiffViewer
      oldValue={oldCode || '(空文件)'}
      newValue={newCode}
      splitView={false}
      showDiffOnly={false}
    />
  )
}
```

**优点**：轻量、无外部依赖、直接可用
**缺点**：不支持流式增量更新（每次要全量刷新）

### 方案 B：Monaco Editor Diff

```bash
npm install @monaco-editor/react
```

```tsx
import DiffEditor from '@monaco-editor/react'

function MonacoDiffView({ original, modified }: { original: string; modified: string }) {
  return (
    <DiffEditor
      original={original || ''}
      modified={modified}
      language="typescript"
    />
  )
}
```

**优点**：真正的 IDE 级 diff、语法高亮、支持流式更新
**缺点**：包体积大（~20MB）、加载慢

### 推荐

初次实现用 **react-diff-viewer**，轻量快速。后续如需更好体验再切换到 Monaco。

## 6. 状态管理

```typescript
interface StreamingFilePreview {
  filePath: string          // 文件路径
  oldContent: string        // 磁盘上当前内容（第一次 read_file 时获取）
  newContent: string        // 正在流式构建的新内容（逐段追加）
  status: 'streaming' | 'written' | 'error'
  errorMessage?: string
  toolCallId?: string       // 关联的 write_file 工具调用
}
```

状态存储位置：

```typescript
// App.tsx 或独立 Context
const [streamingPreviews, setStreamingPreviews] = useState<Map<string, StreamingFilePreview>>({})

// 当收到 text_chunk 且包含文件路径时 → 更新对应预览
// 当收到 tool_call（write_file）时 → 获取原文件内容 = oldContent
// 当收到 tool_result（write_file 成功）时 → status = 'written'
```

---

## 7. 解析策略

### 7.1 识别生成中的代码

LLM 生成的代码通常包裹在 markdown 代码块中：

```
我来修改 `src/auth.ts`：

```typescript
import { defineConfig } from '...

export function setup() {
  // ...
}
```
```

解析规则：

1. 检测 ` 文件名 ` 或 `文件路径` 模式（反引号包裹的路径）
2. 检测 ` ```语言 ` 代码块标记
3. 持续累积代码内容直到代码块关闭
4. 如果关闭前出现了新的 `write_file` 调用，则落盘

### 7.2 关联 write_file

当 SSE 流中收到 `tool_call` 类型事件（`tool_name="write_file"`），从参数中提取 `path`：

```typescript
if (step.stepType === 'tool_call' && step.toolName === 'write_file') {
  const args = JSON.parse(step.content)
  const filePath = args.path
  
  // 为这个文件创建一个新的预览
  // 从磁盘读取当前内容作为 oldContent
  const oldContent = await apiReadFile(filePath)
  addPreview(filePath, oldContent, '')
}
```

后续的 `text_chunk` 如果在该工具调用之前，则累积到对应预览的 `newContent`。

## 8. 实现路线

| 阶段 | 内容 | 工作量 |
|---|---|---|
| **Phase 1** | 选型集成 diff 渲染库 | ~1h |
| **Phase 2** | StreamingDiffPanel 基础组件（文件标签 + diff 视图） | ~3h |
| **Phase 3** | 前端 SSE 事件解析 + 流式内容累积逻辑 | ~2h |
| **Phase 4** | 与工具调用（write_file）的关联联动 | ~1h |
| **Phase 5** | 状态管理（生成中/已写入/错误） | ~1h |
| **Phase 6** | UI 打磨（动画、自动滚动、收合展开） | ~2h |

**总计预估**：约 **10 小时**

---

## 9. 备选方案

如果完整实现成本过高，可以先用简易方案：

### 简易方案：代码块区域高亮

不实现实时 diff，只在聊天框的代码块右侧增加"预览"按钮：

```
┌─────────────────────────────────────┐
│ ┌─────────────────────────────────┐ │
│ │ function hello() {              │ │
│ │   console.log("world")          │ │
│ │ }                               │ │  [📄 预览]
│ └─────────────────────────────────┘ │
└─────────────────────────────────────┘
```

点击后在新标签页打开临时文件内容（不写入磁盘），用户可以提前看到最终内容。

### 对比

| 方案 | 体验 | 工作量 |
|---|---|---|
| 完整 diff 预览 | 优秀——实时看到逐行对比 | ~10h |
| 简易代码块预览 | 一般——只能看不能比 | ~2h |
