## 核心模块

### Agent Runtime

负责执行 Agent 的推理行动循环：
- 解析用户输入
- 调用 LLM 生成响应
- 执行工具调用
- 管理对话状态
### Tools Registry

管理所有可用工具：
- 注册内置工具
- 加载 MCP 工具
- 处理工具调用

### Memory Store

管理短期和长期记忆：
- 存储对话历史
- 管理持久化记忆
- 支持记忆查询

### Skills Loader

加载和管理技能包：
- 解析 ZIP 技能包
- 提取技能配置
- 注册技能到系统

### LLM Client

与 LLM 提供商通信：
- 支持多种提供商
- 处理 API 调用
- 管理重试和超时

## Agent 循环流程

```
用户输入 → 解析 → LLM 推理 → 工具调用 → 结果处理 → 生成响应
                                    │
                              需要确认？ → 用户确认 → 继续执行
```

## 数据流

1. **用户输入** → 前端 → WebSocket → 后端
2. **Agent 处理** → LLM 调用 → 工具执行 → 结果汇总
3. **响应输出** → SSE → 前端 → 用户界面

## 扩展性设计

### 模块化架构

每个模块独立封装，便于扩展和替换：
- 新增工具只需实现工具接口
- 新增 LLM 提供商只需实现客户端接口
- 新增技能只需添加技能包

### 配置驱动

通过配置文件控制行为：
- 无需修改代码即可添加新模块
- 通过配置启用/禁用功能
- 支持多环境配置
## 性能优化

### 异步处理

所有 I/O 操作都采用异步方式，提高并发处理能力。
### 上下文压缩
自动压缩历史消息，减少 LLM 调用成本。
### 缓存机制

缓存常用数据，减少重复计算。
